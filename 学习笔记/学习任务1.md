### 学习任务1 回答
1. 将查询不到的判断放在 Service 层而不是 Controller 层，是为了保持 Controller 的简洁性和职责单一。Controller 主要负责处理 HTTP 请求和响应，而业务逻辑应当放在 Service 层，这样可以更好地复用业务逻辑，并且便于测试。
2. DTO 用于隔离数据库实体和 API 对外契约，避免直接暴露数据库字段、导航属性等内部结构，也方便接口独立演进。HTTP 状态由 ActionResult 表达，统一业务结构由 ApiResponse<T> 表达。
3. 减少重复造轮子，提高复用率，方便后面的维修
4. `ActionResult<ApiResponse<BusinessPartyDto>>` 中的泛型分别代表：
   - `ActionResult`：表示 HTTP 响应的结果类型，可以是成功或失败的响应。
   - `ApiResponse<BusinessPartyDto>`：表示具体的响应数据结构，其中包含了业务数据（`BusinessPartyDto`）以及操作是否成功、错误信息等。
