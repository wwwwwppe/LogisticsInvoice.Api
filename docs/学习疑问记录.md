# 学习疑问记录

本文件持续记录开发过程中的问题、简短解释和后续学习项。明确提问会直接记录；尚未提问但很可能影响理解的关键点，会标注为“建议确认”。

## 2026-06-27｜为什么第一阶段选 SQL Server LocalDB？

- **问题**：MySQL 和 SQL Server 都可以，为什么当前选择 LocalDB？
- **简短解释**：本机已经安装 SQL Server LocalDB，无需安装服务、设置账号或准备 Docker。它与 EF Core SQL Server Provider 配合直接，最有利于第一阶段快速得到可运行结果。
- **后续要补的知识点**：连接字符串、Windows 集成认证、LocalDB 与正式 SQL Server 的差异、如何切换为 MySQL。

## 2026-06-27｜.NET SDK、目标框架和运行时有什么区别？

- **问题**：本机没有 .NET 6 SDK，为什么仍能做 `.NET 6` 项目？
- **简短解释**：项目用 `TargetFramework=net6.0` 决定目标框架；SDK 用于编译；运行时用于执行。本机用 .NET 8 SDK 编译 `net6.0`，最后由已安装的 .NET 6 Runtime 运行。
- **后续要补的知识点**：`global.json`、Target Framework Moniker、SDK 向下兼容、.NET 6 已结束官方支持后的升级考虑。

## 2026-06-27｜为什么客户和供应商使用同一张表？

- **问题**：为什么不分别建立 Customer 和 Supplier？
- **简短解释**：二者在当前阶段字段高度相同，而且同一公司可能既是客户也是供应商。用 `BusinessPartyType` 区分能减少重复代码，并支持 `Both`。如果后期出现大量类型专属字段，再拆分扩展表。
- **后续要补的知识点**：领域建模、表继承方案、过早抽象与适度复用的边界。

## 2026-06-27｜Entity、DTO 为什么要分开？

- **问题**：接口为什么不直接接收和返回 EF Core 实体？
- **简短解释**：Entity 表示数据库结构，DTO 表示接口契约。分开后可以控制可修改字段、隐藏内部字段，并降低数据库结构变化对 API 的影响。
- **后续要补的知识点**：过度提交（Over-posting）、输入 DTO 与输出 DTO、手工映射与 AutoMapper 的取舍。

## 2026-06-27｜Controller、Service、Repository 分别做什么？

- **问题**：三层是否只是把代码拆散？
- **简短解释**：Controller 处理 HTTP；Service 处理业务规则；Repository 处理数据读写。当前结构不追求层数，而是让每类变化有清晰位置，也方便测试和面试说明。
- **后续要补的知识点**：依赖注入、接口的价值、何时可以省略 Repository。

## 2026-06-27｜为什么需要统一返回和全局异常处理？

- **问题**：直接返回对象、在每个接口写 `try/catch` 不行吗？
- **简短解释**：统一结构让前端稳定读取 `success/message/data/errors`；异常中间件集中处理错误、隐藏内部堆栈，并用 `traceId` 对应服务端日志，避免每个接口重复捕获异常。
- **后续要补的知识点**：HTTP 状态码与业务码、Middleware 执行顺序、结构化日志、Problem Details。

## 2026-06-27｜`EnsureCreated` 和 Migration 有什么区别？

- **问题**：为什么第一阶段自动建库，后面还要迁移？
- **简短解释**：`EnsureCreated` 适合快速建立全新数据库，但不擅长管理后续表结构变更；Migration 会记录版本和升级脚本，更适合团队和正式项目。
- **后续要补的知识点**：`dotnet ef migrations add`、`database update`、生产数据库变更审核与回滚。
