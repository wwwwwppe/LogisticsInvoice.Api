### 功能要求

- ID 存在时返回 HTTP 200。
- 响应体必须使用 `ApiResponse<BusinessPartyDto>`。
- ID 不存在时返回 HTTP 404。
- 404 响应仍然使用项目统一错误结构。
- Controller 不得直接访问 Repository 或 `AppDbContext`。
- 不要复制已经存在的“查询不到则抛出 404”逻辑。
- 不要为了完成本题引入 AutoMapper、通用 Repository 或其他新框架。

### 自测步骤

1. 通过 Swagger 新增一条客户数据，记住返回的 ID。
2. 调用 `GET /api/business-parties/{id}`，检查字段和 HTTP 200。
3. 查询一个不存在的 ID，例如 `999999`，检查 HTTP 404 和统一错误结构。
